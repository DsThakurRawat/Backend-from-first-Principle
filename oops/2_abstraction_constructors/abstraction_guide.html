Abstraction & Constructors

This module introduces Abstraction (hiding implementation complexity) and Constructors (object lifecycle initialization).

Definition: Abstraction provides a simplified interface to the user while keeping internal system complexity hidden. This is done in C++ using abstract classes and interfaces.

Encapsulation vs Abstraction

While similar, they have key differences:

  • Encapsulation: Hiding variable states inside the class container to prevent corruption (Data Hiding).
  • Abstraction: Hiding internal system flows (such as hardware queries or math pipelines) and providing simple triggers like start() or deposit() (Implementation Hiding).

Constructors Overview

Constructor Type Signature Pattern Fires When
Default ClassName() Object created with no arguments.
Parameterized ClassName(args...) Object created with specific initial parameters.
Copy Constructor ClassName(const ClassName& other) Object copied (deep or shallow copying).
Destructor ~ClassName() Object goes out of scope or is deleted.
