Polymorphism & Virtual Functions

This module details Compile-time Polymorphism (Function/Operator Overloading) and Run-time Polymorphism (Virtual Functions & Abstract Interfaces).

Definition: Polymorphism translates to "many forms". It allows base interface pointers to invoke correct derived methods dynamically at runtime.

Static vs Dynamic Binding

Feature Compile-time Polymorphism Run-time Polymorphism
Resolution Done at Compilation. Done at Execution runtime.
Mechanism Function Overloading, Operators. Virtual Functions, VTABLEs.
Performance No runtime overhead. Slight overhead for dynamic lookup.

Virtual Destructors Warning

Crucial Rule: Always declare base class destructors as virtual. If omitted, deleting a derived instance through a base pointer will leak derived heap allocations.

Example: Virtual Functions

class Animal {
public:
    virtual void speak() = 0; // Pure virtual
    virtual ~Animal() {}
};
