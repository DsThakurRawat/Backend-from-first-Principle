Encapsulation in C++

Welcome to the ultimate learning module on Encapsulation. Encapsulation is one of the four key pillars of Object-Oriented Programming (OOP) in C++.

Definition: Encapsulation is the process of bundling variables (data) and functions (methods) into a single container called a class, and restricting direct access to prevent corruption.

Why Encapsulation Matters

  • Data Integrity: Prevents direct, unverified access to member variables.
  • Security: Keeps sensitive information hidden from unauthorized modification.
  • Flexibility: Internal details can be updated without affecting parts of the code that use the class.

Access Specifier Mapping

Access Specifier Keyword Accessible From
Public public: Anywhere outside the class.
Protected protected: The class itself and derived classes.
Private private: Only from within the class itself.

Real-World Examples Overview

Example 1: Bank Account System

Demonstrates restricting access to a user's balance. All transactions must be authorized and validated via deposits or withdrawals.

// Access validated via getters and setters
class BankAccount {
private:
    double balance;
public:
    void deposit(double amt) { if (amt > 0) balance += amt; }
};

A student's GPA is read-only from the outside, auto-calculating itself dynamically whenever new grades are registered.
